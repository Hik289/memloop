# Support

Use the GitHub issue templates for product questions, reproducible bugs, and
feature requests.

## Before Opening An Issue

- Run `memloop doctor`.
- Confirm the package version and git commit.
- Reproduce the issue with the smallest local manifest possible.
- Remove private data, generated answers, credentials, and logs before sharing.

## Where To Ask

| Need | Path |
| --- | --- |
| Reproducible bug | Open a bug report with commands and environment details. |
| Product or API improvement | Open a feature request with the blocked workflow. |
| Security concern | Follow `SECURITY.md`; do not post secrets or private data publicly. |
| Release or deployment process | Check `docs/production.md` and `docs/release.md`. |

## Maintainer Checklist

- Ask for `memloop doctor` output when environment details are missing.
- Ask for command lines and redacted manifests, not private corpora.
- Link issues to the docs page that should be updated after the fix.
